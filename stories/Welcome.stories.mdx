# eStaff Design System

Welcome to the **eStaff Design System** documentation. This comprehensive guide showcases our design tokens, components, and patterns that power our freelancer-company marketplace platform.

## üé® Design Philosophy

Our design system is built on three core brand identities:

- **Freelancer (Freela)**: Energetic and creative, using vibrant pink/coral colors
- **Company (Empresa)**: Professional and trustworthy, featuring navy blue tones  
- **Institutional**: Warm and approachable, with beige accents

## üèóÔ∏è Architecture

The design system follows a **token-based architecture** with:

### Design Tokens
- **Colors**: Multi-brand color palettes with semantic naming
- **Typography**: Consistent type scale from `xs` to `9xl`
- **Spacing**: 40-point spacing scale for consistent layouts
- **Animation**: Premium motion design with 60fps animations
- **Elevation**: Layered shadow system for depth

### Component Hierarchy
1. **Primitives**: Basic building blocks (Button, Input, Badge)
2. **Elements**: Composed components (Card, Form, Modal)
3. **Patterns**: Complex UI patterns (Navigation, DataTable)
4. **Templates**: Full page layouts

## üöÄ Getting Started

### Installation

```bash
npm install
npm run storybook
```

### Using Components

```tsx
import { Button } from '@/components/ui/button'

<Button variant="primary" size="lg" brand="freela">
  Get Started
</Button>
```

### Applying Tokens

```tsx
import { colors, spacing, typography } from '@/lib/design-system/tokens'

const styles = {
  color: colors.freela[500],
  padding: spacing[4],
  fontSize: typography.sizes.lg,
}
```

### Theme Switching

All components are WCAG 2.1 AA compliant with:

- **Bundle Size**: <500KB initial load
- **Core Web Vitals**: Optimized for LCP, FID, and CLS

Mobile-first approach with breakpoints:
- Mobile: 375px
- Tablet: 768px
- Desktop: 1366px
- Wide: 1920px
